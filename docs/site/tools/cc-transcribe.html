Overview

cc-transcribe is a powerful video transcription tool that uses OpenAI Whisper to generate timestamped transcripts. It also intelligently extracts screenshots when visual content changes, making it perfect for processing tutorials, presentations, and lectures.

Requires: OpenAI API key and FFmpeg installed on your system.

Features

  • Transcribe any video format (mp4, mkv, avi, mov)
  • Timestamped transcript output
  • Automatic screenshot extraction at content changes
  • SSIM-based visual change detection
  • Configurable sensitivity and intervals
  • JSON and plain text output formats

Requirements

OpenAI API Key

Set environment variable
# Windows
set OPENAI_API_KEY=your-key-here

# macOS / Linux
export OPENAI_API_KEY=your-key-here

FFmpeg

FFmpeg must be installed and accessible in your PATH:

Install FFmpeg
# Windows (Chocolatey)
choco install ffmpeg

# macOS (Homebrew)
brew install ffmpeg

# Linux (apt)
sudo apt install ffmpeg

Usage

Basic Syntax
cc-transcribe VIDEO_FILE [OPTIONS]

Options

Option Default Description
-o, --output Auto Output directory (default: {input}_transcript/)
--screenshots/--no-screenshots Enabled Extract screenshots at content changes
-t, --threshold 0.92 Screenshot sensitivity (0-1, lower = more)
-i, --interval 1.0 Minimum seconds between screenshots
-l, --language Auto Force language code (en, es, de, etc.)
--info - Show video info and exit
-v, --version - Show version and exit

Output Files

cc-transcribe creates a folder with the following structure:

Output directory structure
video_transcript/
transcript.txt # Timestamped transcript
transcript.json # Full JSON with segments
screenshots/ # Extracted images
00-00-05.png
00-01-23.png
00-02-45.png
...

Transcript Format

The text transcript includes timestamps in [MM:SS] format:

transcript.txt example
[00:00] Welcome to this tutorial on Python programming.
[00:05] Today we'll cover the basics of functions.
[00:12] Let's start by looking at how to define a function.
[00:18] Here's a simple example...

Examples

Basic Transcription

Transcribe with default settings
cc-transcribe meeting.mp4

Custom Output Directory

Specify output location
cc-transcribe lecture.mp4 -o ./transcripts/lecture

Transcript Only (No Screenshots)

Skip screenshot extraction
cc-transcribe podcast.mp4 --no-screenshots

More Screenshots

Lower threshold for more captures
cc-transcribe tutorial.mp4 -t 0.85 -i 0.5

Force Language

Specify transcription language
cc-transcribe video.mp4 -l es

How It Works

cc-transcribe extracts audio for transcription while simultaneously analyzing frames for content changes

Processing Steps

  1. Extract Audio: Uses FFmpeg to extract the audio track from the video
  2. Transcribe: Sends audio to OpenAI Whisper API for transcription with timestamps
  3. Analyze Frames: Reads video frames and compares using SSIM (Structural Similarity Index)
  4. Extract Screenshots: Saves frames where visual content has changed significantly
  5. Generate Output: Creates timestamped transcript files and organized screenshot folder

Screenshot Detection

The tool uses SSIM (Structural Similarity Index) to compare consecutive frames. When similarity drops below the threshold, a screenshot is captured. This is particularly effective for:

  • Slide presentations
  • Code tutorials with screen changes
  • Software demonstrations
  • Lecture recordings with slides
