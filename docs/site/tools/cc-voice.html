cc-voice

Convert text to natural-sounding speech with multiple voices and automatic markdown cleaning.

Overview

cc-voice converts text to natural-sounding speech using OpenAI's Text-to-Speech API. It supports multiple voices, automatic markdown cleanup, and handles long texts by splitting them into manageable chunks.

Requires: OpenAI API key set as OPENAI_API_KEY environment variable.

Features

  • 6 distinct voices to choose from
  • Standard and HD quality options
  • Automatic markdown formatting removal
  • Adjustable speech speed (0.25x to 4x)
  • Handles long texts with automatic chunking
  • Accepts text directly or from files

Requirements

Set API Key
# Windows
set OPENAI_API_KEY=your-key-here

# macOS / Linux
export OPENAI_API_KEY=your-key-here

Usage

Basic Syntax
cc-voice TEXT_OR_FILE -o OUTPUT.mp3 [OPTIONS]

The first argument can be either:

  • Direct text in quotes
  • Path to a text file (.txt, .md, etc.)

Options

Option Default Description
-o, --output Required Output audio file path (.mp3)
-v, --voice onyx Voice: alloy, echo, fable, nova, onyx, shimmer
-m, --model tts-1 Model: tts-1 (standard), tts-1-hd (high quality)
-s, --speed 1.0 Speed: 0.25 to 4.0
--raw false Don't clean markdown formatting
--version - Show version and exit

Available Voices

OpenAI provides 6 distinct voices, each with its own character:

Voice Description
alloy Neutral, versatile voice
echo Warm, resonant male voice
fable British-accented narrator voice
nova Friendly, conversational female voice
onyx Deep, authoritative male voice (default)
shimmer Clear, bright female voice

Models

  • tts-1 - Standard quality, faster, lower cost
  • tts-1-hd - High definition, better for professional use

Examples

Simple Text

Convert text directly
cc-voice "Hello, welcome to CC Tools!" -o greeting.mp3

From File

Convert a text file
cc-voice article.txt -o article.mp3

Different Voice

Use nova voice
cc-voice script.md -o narration.mp3 -v nova

High Quality

HD quality with fable voice
cc-voice story.txt -o audiobook.mp3 -v fable -m tts-1-hd

Adjust Speed

Slower speech for clarity
cc-voice tutorial.txt -o slow_tutorial.mp3 -s 0.8

Raw Markdown

Keep markdown symbols
cc-voice notes.md -o notes.mp3 --raw

How It Works

cc-voice cleans markdown, splits long text into chunks, and converts each chunk to speech

Processing Steps

  1. Read Input: Loads text from argument or file
  2. Clean Markdown: Removes formatting symbols (# headers, ** bold, etc.) unless --raw is used
  3. Split Chunks: Breaks long text into 4096-character chunks at sentence boundaries
  4. Call TTS API: Sends each chunk to OpenAI TTS with selected voice and model
  5. Combine Audio: Merges all audio chunks into the final output file

Markdown Cleaning

By default, cc-voice removes markdown formatting for natural speech:

  • Removes # Header markers
  • Removes **bold** and *italic* markers
  • Converts [links](url) to just the link text
  • Removes code fence markers
  • Preserves actual content for natural reading
